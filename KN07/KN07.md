# KN07: PAAS

Beachten Sie die [allgemeinen Informationen zu den Abgaben](https://gitlab.com/ch-tbz-it/Stud/m346/m346/-/blob/main/Abgaben.md).



## Lernziele

- Sie verwenden das Servicemodell PAAS um eine Datenbank zu hosten.
- Sie arbeiten mit Serverless Funktionen arbeiten (FaaS) und setzen dabei auch praktische Wartungsarbeit um.
- Sie arbeiten mit Serverless Funktionen arbeiten (FaaS) und greifen auf eine Datenbank zu.
- Sie verwenden das Servicemodell PAAS um eine Applikation zu hosten.
- Sie vergleichen die Objekte, die im Hintergrund automatisch erstellt wurden und sehen so was PAAS Ihnen im Vergleich zu IAAS abnimmt.



## A) Datenbank im PAAS Modell (20%)

Erstellen Sie eine Datenbank und verwenden Sie dazu den AWS Service *RDS*.  Im Gegensatz zu MongoDB Atlas aus KN06 ist AWS RDS ein PAAS Service.

Folgende Einstellungen sind wichtig, schauen Sie die anderen ebenfalls durch:

- Datenbanktyp: Mariadb (einfach nur damit wir kompatibel zu den bisherigen Aufträgen bleiben)
- Verwenden Sie den *Free Tier*. 
- Verwenden Sie ein Master Password. **Wichtig**: Verwenden Sie die beiden Zeichen [#,-] (Raute und Bindestrich) **nicht**, weil dies zu Verbindungsproblemen führen kann. 
- Stellen Sie sicher, dass die Sicherheitsgruppe korrekt ist
- Geben Sie Ihrer Datenbank eine öffentliche IP.

Testen Sie nun die Verbindung mit MySqlWorkbench und führen Sie das gleiche Query wie aus der Datei db.php aus.

**Abgaben:**

- Screenshot aus MySqlWorkbench mit dem ausgeführtem Query
- Erklären Sie wieso Sie besser einen PAAS oder SAAS Service verwenden anstatt eine eigene Datenbank zu installieren



## B) PAAS Applikation erstellen (60%)

Verwenden Sie [AWS Elastic Beanstalk](https://docs.aws.amazon.com/elasticbeanstalk/latest/dg/Welcome.html) um eine Applikation zu erstellen. Sie können eine Beispiel-Applikation von AWS (im GUI integriert), eine eigene Applikation oder eine frei verfügbare Applikation (z.B. [Spring PetClinic](https://github.com/spring-projects/spring-petclinic)) verwenden. 

Gehen Sie weiter durch die einzelnen Seiten und füllen Sie die Inhalte sinnvoll aus, z.B.

- Verwenden Sie einen LoadBalancer
- Fügen Sie das Monitoring hinzu
- etc

Für die **veränderten Bereiche** erstellen Sie einen Screenshot und dokumentieren Sie was und wieso Sie die Einstellungen so gewählt haben. 

**Abgaben**

- Screenshots für die veränderten Bereiche
- Erklärungen zur Auswahl in den verschiedenen Screenshots.
