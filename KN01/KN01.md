# KN01: Virtualisierung

Beachten Sie die [allgemeinen Informationen zu den Abgaben](https://gitlab.com/ch-tbz-it/Stud/m346/m346/-/blob/main/Abgaben.md).

In dieser Kompetenz arbeiten Sie mit einer lokalen Virtualisierungsumgebung. Sie sollen ein Gefühl für den Umgang mit Virtualisierung und Installation von Linux kriegen.  

## Lernziele

- Sie machen die ersten Schritte Virtualisierung 

- Sie lernen die Grenzen der Virtualisierung kennen

  

## A) Hypervisor Typ 1 und 2 (30%)

Suchen Sie im Internet nach den beiden Begriffen Hypervisor 1 und Hypervisor 2. Erklären Sie was ein Hypervisor ist und den Unterschied der beiden Typen.

**Abgabe:**

- Ein paar Sätze zu dem Begriff und den Unterschieden



## B) Virtualisierungssoftware (70%)

Tun Sie sich in Gruppen zusammen (2-4) und stellen Sie sicher, dass einer von Ihnen bereits eine Virtualisierungssoftware im Einsatz hat (vmware, virtualbox, multipass, etc - aber kein Docker). Dies kann etwas lokales sein oder eigene Server mit dem Betrieb einer Virtualisierung. 

Stellen Sie nun eine Vermutung an, ob Ihr System ein Hypervisor Typ 1 oder Typ 2. 

Testen Sie Ihre Vermutung, indem Sie mit der Ressourcenzuweisung arbeiten (RAM und CPU), z.B.

- Überprüfen Sie zuerst wie viel Logische Prozessoren und RAM Ihr **Host-System** hat, also ihr eigentliches Betriebssystem.
- Erstellen Sie eine virtuelle Maschine (z.B. Ubuntu, Debian)
- Weisen Sie Ihrer VM **mehr** Prozessoren zu, als Ihr Host-System logische Prozessoren hat. Starten Sie die VM und rufen Sie in der Konsole den Befehl  `lscpu | grep "CPU(s)"` auf.  (Screenshot für Abgabe nicht vergessen). Es ist möglich, dass Sie nicht mehr Prozessoren hinzufügen können als Ihr Host-System hat. Dann machen Sie den Screenshot der entsprechenden Fehlermeldung oder Einschränkung.
- Machen Sie die gleichen Schritte wie eben mit der Ressource **RAM**. Der Linux Befehl zum Auslesen ist `free -h. Es ist möglich, dass Sie nicht mehr RAM hinzufügen können als Ihr Host-System hat. Dann machen Sie den Screenshot der entsprechenden Fehlermeldung oder Einschränkung.

**Abgaben**

- Kurzer Text zur Vermutung des Hypervisor Typs Ihres Systems
- Dokumentieren Sie mit **Screenshots** Ihre Einstellungen, Resultat beim Auslesen der Ressource und/oder Fehlermeldungen
- **Erklärung** wieso Sie Fehlermeldungen kriegen oder Erklärung wie es möglich ist, dass Sie mehr Ressourcen zuteilen können als Ihr Host-System hat. **Machen Sie sinnvolle Überlegungen, nicht einfach nur Beobachtungen**. Hat sich Ihre Vermutung bestätigt?
